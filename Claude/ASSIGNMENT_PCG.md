## Prozedurale Erweiterung der Spielwelt (K3, S2, S3)

Entwickle Scripts und UI-Elemente, die es dir im Editor ermöglichen
prozedural generierte Level zu erstellen bzw. Teilaspekte der
Level-Erstellung zu automatisieren. Der Grundaufbau des Levels soll dabei
mit passenden Algorithmen per Code erstellt werden. Als Fallbeispiele für
die Level eignen sich Landschaften oder Dungeons. Hierzu gibt es viele
passende Algorithmen, die du ausprobieren kannst. Dein Tool sollte
diverse Einstellungsmöglichkeiten bieten, um so verschiedenste Ergebnisse
zu erzeugen. Integriere dieses Tool in dein Spielprojekt.

Wähle passende Features aus den folgenden Möglichkeiten für dein Tool.
Hierzu ein paar Vorschläge:

- Dungeon/Innen-Level
  - Größe des Levels (evtl. unendlich groß)
  - Unterschiedliche Level-Typen (Räume, Gänge, Labyrinthe, etc.)
  - Platzierungsmöglichkeiten für NPCs, Items, etc.
  - Verschiedene Grafiksets

- Landschaft/Terrain
  - Größe des Levels (evtl. unendlich groß)
  - Beschaffenheit der Landschaft (hügelig, flach, bergig, etc.)
  - Unterschiedliche Biome (Wüste, Wald, Tundra, Graslandschaft, etc.)
  - Texturierung der Landschaft / bzw. der Biome
  - Plateaus auf verschiedenen Ebenen
  - Platzierungsmöglichkeiten für Gebäude, Pflanzen, etc.

Du kannst dir aber auch gerne andere Anwendungsmöglichkeiten überlegen.
Wenn du dir unsicher bist, ob deine Idee zur Aufgabe passt oder
umfangreich genug ist, besprich diese mit deinem Fachbereichs-Team.

### Tipps
- Wende dich bei Problemen an dein Fachbereichs-Team.
- Lass andere aus deinem Kurs deinen Code testen, um mögliche Fehler zu
  finden.
- Starte mit simplen Konfigurationen und verfeinere diese im Verlauf der
  Aufgabe.
- Probiere verschiedene Algorithmen zum Erstellen und Verformen von
  Meshes aus.
- Probiere verschiedene Algorithmen zur Level-Generierung aus.
- Versuche die Abläufe beim Erstellen des Levels zu automatisieren, damit
  du möglichst wenig "Handarbeit" hast.

### Lernziele
- K3 — Unterscheiden zwischen verschiedenen Algorithmen zur Generierung
  von Inhalten
- S2 — Erstellen von generierten Inhalten für Spielewelten
- S3 — Erstellen einer Simulation einer gewohnten Umgebung mit einer
  generierten Bevölkerung

### Feedbackelemente
Proficiency:
- Läuft der Code stabil und fehlerfrei?
- Wird das Mesh effektiv und performant erzeugt?
- Ist das Mesh stark einstellbar / modular?

Process:
- Wurde die vorgegebene Coding-Convention verwendet?
- Werden Code-Abschnitte sinnvoll getrennt und verwaltet?
- Ist die Topologie des Meshes passend für die Aufgabe?
- Wurde im Mesh auf Besonderheiten der Spielewelt Rücksicht/Bezug
  genommen?

Person:
- Ist der Code kommentiert und lesbar?
- Wurden mehrere Einstellungsmöglichkeiten / Arten von Terrain
  implementiert?
- Wurde sich genügend Feedback zur Abgabe geholt und dieses umgesetzt?
